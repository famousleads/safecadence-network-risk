# v9.38 — Small pages audit

Same audit-then-fix exercise as v9.33 / v9.35 / v9.36 / v9.37. The
"small pages" group covers the surfaces that aren't the headline
features but operators touch daily: timeline, share, automation,
watchlists, briefing, settings, audit log.

**Result: clean.** All seven surfaces are real and wired. Two minor
transparency / UX items shipped in this release.

## Strong trust posture

- **Share tokens auto-expire.** `create_share()` enforces TTL (1/7/30
  days). Revoke endpoint works. Public dashboard is read-only by
  design and shows scope plainly.
- **Automation rate-limited.** Default 3600s between fires per rule.
  Prevents runaway notifications.
- **Timeline window-bounded.** Max 200 events per response, default
  24h window. Operators can't accidentally OOM the page.
- **Settings persist to disk; restart required.** Honest about scope —
  no hot-reload illusion that drifts from reality.
- **Audit log combines both sources.** Policy audit (JSONL) and
  execution audit (SQLite) surface in one filtered view.

## Findings

### #1 — `/automation` preview doesn't say "dry-run" loudly

The preview button POSTs to `/api/intel/automation/preview` which
correctly calls `evaluate_rules(findings, apply_actions=False)` — no
actions fire. But the UI just dumps the JSON in a `<pre>` block
without telling the operator what they're looking at. Someone
scanning quickly could misread "would_fire" → "fired".

**Severity:** transparency gap. Trust posture is correct (no actions
fire), the labeling is what's wrong. Two-line fix.

**Fix:** prefix the preview output with "🔮 Preview only — no
actions taken. These are the rules that *would* fire against current
findings." Wrap in a banner so it's hard to miss.

### #2 — `/timeline` kinds filter is a freeform text box

Operator types `automatin` (typo), gets empty results, thinks no
automation events happened. The kinds field needs a `<datalist>` with
the actual kinds the timeline emits.

**Severity:** UX friction. Park-able, but the fix is small enough
to ship in v9.38.

**Fix:** populate a `<datalist>` from the static set
`audit | jit | comment | assignment | watchlist | automation` so
typos auto-correct and operators see what's available.

### #3 — `/briefing` has no progress indicator for long generations

Button shows "Generating…" but on a 100-asset fleet the briefing
takes 10-15s. No spinner means operator wonders if it hung.

**Severity:** cosmetic. Park for v9.39 polish.

### #4 — Audit log timestamps are raw ISO 8601

A 2-week-old entry renders as `2026-04-22T14:30:00Z` instead of "2
weeks ago". Operators have to do mental arithmetic.

**Severity:** cosmetic. Park.

### #5 — Settings writes have no schema validation

`POST /api/settings/*` writes whatever JSON is sent. An operator can
break their next restart with bad config. Self-inflicted damage
only — requires direct API access.

**Severity:** low. Park to a future release that introduces a
settings schema.
