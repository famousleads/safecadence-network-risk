Same audit-then-fix exercise we ran on Identity (v9.33-9.34) and Execute
(v9.35.0). The Discover section covers ten surfaces; eight are mature and
production-ready, one is mostly complete with a documented heuristic, and
**one HTTP endpoint is a fake-success** that violates the trust posture.

## Strong trust posture

- **Daemon best-effort hooks.** `daemon.run_cycle` calls
  `_run_due_discovery_jobs()` inside a `try/except` so one failed source
  never aborts the cycle. Per-source runners (`lan-scan`, `snmp`, `ad`,
  `entra`, `dhcp`, `aws`, `azure`, `gcp`) each return `(ok, error)`.
- **Asset write surfaces are gated.** `/api/platform/asset` (manual add)
  and `/api/platform/discovery-jobs` (CRUD) require `require_writer`
  dependency. Reads use `get_current_user`.
- **Topology has empty-state guidance.** When `node_count == 0`, the
  page shows "Run SNMP harvest first" with a deep link instead of a
  blank canvas.
- **Coverage shows source-by-source health.** Each connector renders
  with `fresh / stale / very-stale / offline` and an inline CTA so the
  operator can act without leaving the page.

## Findings

### #1 — `/api/platform/discovery-jobs/{id}/run-now` is a fake-success **(trust violation)**

`src/safecadence/server/platform_api.py:290-309`

```python
@app.post("/api/platform/discovery-jobs/{job_id}/run-now")
def discovery_jobs_run_now(job_id: str, user=Depends(require_writer)):
    j = get_job(job_id)
    if not j:
        raise HTTPException(404, detail="job not found")
    # We can't import the FastAPI app's other endpoints here cleanly,
    # so just stamp the run as 'ok' — the UI will redirect to the
    # actual discovery hero card on /inventory which already does
    # the work.
    mark_run(job_id, ok=True)
    return {"status": "ok", "redirect": "/inventory", "hint": "..."}
```

The endpoint marks the job as a successful run *without ever firing the
discovery*. The UI shows a green checkmark; the operator believes the
job ran; the fleet is unchanged. This is exactly the "looks real but
isn't" pattern v9.33 / v9.35 fixed elsewhere.

The daemon path is correct — `daemon._fire_discovery_job(j)` dispatches
to the right runner per source. The HTTP endpoint should call the same
function instead of stubbing.

**Fix:** extract `_fire_discovery_job` into a callable in
`intel/discovery_jobs.py`, then have both the daemon and the HTTP
endpoint call it. The endpoint becomes:

```python
ok, err = fire_job(j)
mark_run(job_id, ok=ok, error=err)
return {"status": "ok" if ok else "error",
        "ok": ok, "error": err}
```

### #2 — Discovery job params have no schema validation

`src/safecadence/server/platform_api.py:265-280`

```python
j = create_job(name=..., source=..., params=body.get("params") or {}, ...)
```

`params` is whatever JSON the operator submits. Failure mode: invalid
params (`cidr` missing for `lan-scan`, `host` missing for `snmp`) are
caught only at run time, returning `(False, "lan-scan requires
params.cidr")`. The operator never sees this until the next cycle.

**Fix:** add `validate_params(source, params) -> (ok, error)` in
`intel/discovery_jobs.py` that knows the required keys per source
(matching the runner branches). Call it from `create_job` and raise
`ValueError` so the HTTP endpoint returns 400 with the exact missing
key.

### #3 — `/scope` ships only 3 hardcoded compliance frameworks

The "Add scope" wizard renders SOC2, PCI, HIPAA buttons. Assets
already tagged with other `compliance:*` values render in the detail
list but can't be re-tagged or used as wizard targets.

**Severity:** UX gap, not a trust violation. Park for v9.38 (small
pages) unless the fix is one-line.

### #4 — `/coverage` recommendation ranking is opaque

`compute_coverage()` returns `recommendations: [{priority, title,
action_url}]` but the priority numbers are baked into the function with
no documentation. Operators can sort but can't argue with the order.

**Fix:** add a `reason` field per recommendation. Two-line change in
`intel/coverage.py`.

### #5 — Discovery job source dropdown is hardcoded in the UI

`_JOBS_BODY` lists `lan-scan / snmp / ad / entra / dhcp / aws / azure /
gcp` as `<option>` tags. If a new source is added to the daemon
runner, the UI silently drops it.

**Fix:** expose `GET /api/platform/discovery-jobs/sources` returning
the list from a single source-of-truth constant in
`intel/discovery_jobs.py`. The UI populates the dropdown from that.
